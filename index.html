Смещения и нули

Fanuc

Смещения нуля детали

6 стандартных смещений: G54–G59. Расширенные: G54.1 P1–P48 (или G54.1 P1–P300 в зависимости от опции).

Задаются через меню OFFSET / WORK или командой: G10 L2 P1 X... Y... Z...

Пример
G54          ; активировать смещение 1
G10 L2 P1 X-120.5 Y-85.0 Z0 ; записать G54 из программы
Siemens

Нулевые смещения (ZO)

G54–G57 + расширенные G505–G599 (FRAME-функции). Программируются через TRANS/ATRANS (трансляция) и ROT/AROT (поворот).

Возможна работа с составными FRAME: FRAME = CTRANS(X, v1, Y, v2)

Пример
G54              ; активировать нулевое смещение
TRANS X50 Y30    ; дополнительный сдвиг
ATRANS X10       ; накопительный сдвиг
Heidenhain

Нулевые точки (Preset / REF)

Используются таблицы нулевых точек (Preset-таблица) или цикл CYCL DEF 7. Активация через CYCL DEF 7.0 / 7.1 / 7.2 / 7.3.

Каждый параметр цикла задаёт смещение по одной оси.

Пример
CYCL DEF 7.0 НУЛЕВАЯ ТОЧКА
CYCL DEF 7.1 X+50
CYCL DEF 7.2 Y+30
CYCL DEF 7.3 Z+0
; или через таблицу:
CYCL DEF 7.0
CYCL DEF 7.1 #5  ; строка 5 таблицы

Коррекция инструмента

Fanuc

Таблица офсетов (H/D)

  • H-офсет — длина инструмента (G43 H...)
  • D-офсет — радиус инструмента (G41/G42 D...)
  • Активируется через G43: G43 H1
  • Отмена: G49
  • Износ хранится отдельно в колонке Wear
Пример вызова
T01 M06          ; сменить на инструмент 1
G43 H01 Z100.   ; активировать коррекцию длины
G41 D01 X...    ; коррекция радиуса влево
Siemens

Таблица инструментов TO

  • Коррекция автоматически при вызове T + M6
  • D1, D2… — режущие кромки одного инструмента
  • Длина по Z задаётся как Length1 в таблице
  • Радиус — Radius в таблице TO
  • Смена кромки: D2 (без смены T)
Пример вызова
T1 D1            ; инструмент 1, кромка 1
M6               ; смена инструмента
G41              ; коррекция радиуса влево (из таблицы)
Heidenhain

Таблица инструментов TOOL.T

  • Все данные — в файле TOOL.T (Length, Radius, DL/DR — износ)
  • Вызов: TOOL CALL n Z S5000
  • Коррекция радиуса: RL / RR / R0 в кадре движения
  • Отдельный файл TOOL_P.T для карманов
  • Ось инструмента задаётся при вызове: X / Y / Z
Пример вызова
TOOL CALL 5 Z S6000  ; инструмент 5, ось Z, 6000 об/мин
L X+50 Y+30 RL F300  ; движение с коррекцией радиуса влево

Опасные различия — читать внимательно!

⚠

Компенсация длины инструмента

На Fanuc коррекция длины не активна без явного G43 H.... На Siemens и Heidenhain она применяется автоматически при вызове инструмента. Если перенести программу Fanuc на Siemens без удаления G43 — программа может не запуститься или отработать с ошибкой.

⚠

Единицы: G20/G21 vs G70/G71

Fanuc: дюймы = G20, метрика = G21. Siemens: дюймы = G70, метрика = G71. Если перепутать — инструмент уйдёт в 25 раз дальше или ближе нужной позиции. Катастрофический риск столкновения.

⚠

Отмена циклов сверления

Fanuc: цикл продолжает действовать до явного G80. Siemens и Heidenhain: цикл однократный или требует MCALL. Без G80 на Fanuc любое движение XY может вызвать незапланированное сверление.

⚠

Вызов подпрограммы (M98/CALL)

Fanuc: M98 P1001 L3 (номер + повтор). Siemens: SUB1() или CALL "SUB1". Heidenhain: CALL PGM "путь". При переносе программы подпрограммы просто не вызовутся без ошибки — опасная ситуация.

⚠

Направление дуги у Heidenhain

Heidenhain использует DR+ (против часовой) и DR- (по часовой) — это обратная логика по сравнению с G02/G03. При конвертации постпроцессора легко перепутать направление контурной фрезеровки.

⚠

Смена инструмента: T и M06

Fanuc: T01 M06 — сначала подготовка (T), потом физическая смена (M06). Если написать только T01 без M06 — инструмент не сменится. На Siemens поведение зависит от конфигурации PLC — может сменяться при T или при M6.

⚠

Конец программы: M30 vs END PGM

Heidenhain требует строку END PGM имя MM в конце каждой программы. Без неё программа не верифицируется. Fanuc/Siemens: M30 — обязательно для сброса указателя. Без M30 на Fanuc программа может запуститься повторно с середины.

⚠

Инкрементальный режим Heidenhain

Heidenhain: для инкрементального перемещения используется префикс I перед адресом оси: L IX+10. Fanuc/Siemens: переключение режима через G91. Смешивание логик — частая ошибка при конвертации программ.

; SYM-  — выбор «короткого» решения поворота
; TURN  — шпиндель разворачивается перп. плоскости

; --- АКТИВАЦИЯ TCPM ---
M128 F3000          ; TCPM вкл, макс. подача компенсации 3000

; --- ОБРАБОТКА В НАКЛОННОЙ ПЛОСКОСТИ ---
L X+0 Y+0 R0 FMAX
L Z+5 R0 FMAX
CYCL DEF 200 СВЕРЛЕНИЕ
  Q200=+2    ; безоп. расстояние
  Q201=-15   ; глубина
  Q206=+150  ; подача врезания
  Q202=+5    ; шаг врезания
  Q210=+0    ; выдержка вверху
  Q203=+0    ; поверхность
  Q204=+50   ; 2-е безоп.расст.
  Q211=+0    ; выдержка внизу
L X+0 Y+0 FMAX M3 CYCL CALL
L X+30 Y+20 FMAX CYCL CALL

; --- СБРОС ---
L Z+200 R0 FMAX
M129                ; TCPM выкл
PLANE RESET MOVE FMAX   ; сброс наклона
END PGM TILT5AX MM
💡
Важно: TCPM/RTCP требует лицензии! На Heidenhain — опция 8 (TCPM), на Siemens — опция TRAORI, на Fanuc — опция 5-Axis Machining. Без лицензии команды активируются, но компенсация не работает — инструмент уйдёт в неверную точку.

Привязка нуля детали вручную (без циклов привязки)

Самая частая операция перед запуском. Нулевая точка говорит станку: «вот откуда считать X=0, Y=0, Z=0».

Fanuc — G54
  1. 1
    Касаемся датчиком/инструментом точки X=0 на детали
  2. 2
    OFS/SET → Измерение → G54
    Вводим текущую координату X станка, нажимаем MEASURE или вводим вручную
  3. 3
    Повторяем для Y, потом Z (обычно верх детали)
  4. 4
    В программе вызываем: G54 G90
⚠️ G54 — первая из 6 рабочих систем (G54–G59). Убедись что программа вызывает именно ту, куда записал нуль.
Siemens — WO (Рабочий ноль)
  1. 1
    Касаемся точки X=0 на детали
  2. 2
    Offset → WO → G54
    Вводим значение X, нажимаем Задать нулевую точку
  3. 3
    Повторяем для Y и Z
  4. 4
    В программе: G54 G90 — так же как Fanuc
💡 На 828D интерфейс может отличаться, но логика та же — найди пункт меню «Нулевые смещения» или «WO».
Heidenhain — Preset
  1. 1
    Переходим в режим Ручное управление
  2. 2
    Касаемся точки X=0 на детали
  3. 3
    Preset → Ввести значение
    Вводим 0 для X, подтверждаем
  4. 4
    Повторяем для Y, Z
  5. 5
    В программе: CYCL DEF 7.0 DATUM SHIFT или просто Preset №0 активен автоматически
💡 На TNC 640 таблица Preset удобнее — можно хранить несколько нулей и переключаться между ними.

FAQ — частые вопросы оператора

Реальные вопросы с производства. Простые ответы — без лишней теории.

🔧 Подготовка и наладка 9 вопросов
  • Уровень масла, СОЖ, давление воздуха — по приборам на станке
  • Чистота рабочей зоны, стружкоуборка, защитные шторки
  • Нет ли посторонних предметов в зоне обработки
  • Отработка референтного хода (HOME) если станок был обесточен
  • Проверить таблицу коррекций — не менял ли кто в ночную смену
  • Прогрев шпинделя по циклу станка (обычно 5–10 минут) если долгий простой
  • Правильное имя и версия программы в стойке (путаница версий — частая причина брака)
  • Нулевые точки G54–G59 соответствуют карте наладки
  • Инструменты по номерам совпадают с картой: тип, диаметр, вылет
  • Коррекции длины и радиуса инструментов введены и не нулевые там, где не должны быть
  • Заготовка зажата правильно, упоры и базы как по карте
  • Первый запуск — только с подачей 5–10% и рукой на «Стоп»

Когда обязательно: новая программа впервые, после правок в программе, после смены оснастки или базирования, после долгого простоя станка.

Как делать: подними Z на 50–100 мм выше детали (смещением нуля по Z или функцией «приподнять» на стойке), запусти на 5% подачи, проверь что траектория не цепляет зажимы и деталь. Потом — первый проход с подачей 10–20% и рукой у «Стоп».

Достаточно когда: ты видел все переходы, все смены инструмента прошли нормально, нет подозрительных сближений с оснасткой.

Задаёт наладчик/технолог — при первой наладке операции. Записывается в карту наладки.

Оператор может: корректировать нуль в пределах ±0.1–0.5 мм для подстройки по результату первой детали (если это предусмотрено картой).

Оператор не должен: менять нуль «на глазок» без измерения, менять систему нулей (G54 на G55), переписывать нули без записи в журнал.

⚠️ Любое изменение нуля — записать в журнал. Иначе следующая смена получит брак и не поймёт почему.
  • Индикатором проверить параллельность/перпендикулярность губок тисков столу (допуск — по карте, обычно 0.02–0.05 мм)
  • Все болты и гайки затянуты с нужным моментом (не «на руку»)
  • Заготовка опирается на все три базовые точки, нет качания
  • После зажима — тихонько стукни деталь молотком через прокладку: не должна сдвинуться
  • Упоры стоят как по чертежу приспособления, не заменены на «что было под рукой»
  • Геометрия станка — биение шпинделя, перпендикулярность осей (ремонтник должен дать протокол)
  • Все параметры станка на месте — бэкап параметров сравнить с эталонным
  • Коррекции инструментов и нули — проверить по эталонной детали или контрольному образцу
  • Прогрев шпинделя по расширенному циклу (15–30 мин)
  • Первую деталь — обязательно промерить полностью по карте контроля
  • Необычные звуки или вибрация от шпинделя, приводов, направляющих
  • Тепло от шпинделя или осей больше обычного
  • Люфт в осях, который раньше не наблюдался
  • Масло или СОЖ текут не туда, где должны
  • Повторяющиеся аларм-коды которые не исчезают после перезапуска
  • После столкновения — даже если «кажется всё нормально»
⚠️ «Дотянуть до конца смены» после столкновения — самая дорогая ошибка. Скрытые повреждения шпинделя проявляются через 2–3 часа работы, и ремонт тогда в разы дороже.
  1. Сверить программу — имя, версия, последнее изменение
  2. Промерить вылет каждого нового инструмента, записать в таблицу коррекций
  3. Проверить нуль детали по базовой поверхности
  4. Сухой прогон без детали
  5. Первый запуск с заготовкой на подаче 10%, рука на «Стоп»
  6. Промер первой детали — 100% размеров по карте контроля
  • Правило 3-2-1: основная плоскость — 3 точки, направляющая — 2, упор — 1
  • Базовые поверхности должны быть чистыми — стружка под деталью = смещение нуля и брак
  • Зажим — после установки на базы, не наоборот
  • Усилие зажима: достаточно чтобы деталь не сдвинулась от сил резания, но не так чтобы деформировать заготовку
  • Тонкостенные детали — обязательно поддержка снизу, иначе вибрация и брак по шероховатости
🔩 Инструмент и режимы 8 вопросов

Технолог/программист — выбирает тип инструмента, марку, геометрию, задаёт базовые режимы (Vc, fz, ap, ae) в карте наладки и программе.

Оператор-наладчик — контролирует соответствие реального инструмента карте, вводит коррекции, следит за износом, корректирует подачу в разумных пределах если что-то пошло не так.

💡 Если реального инструмента нет и ты хочешь поставить «похожий» — согласуй с технологом. Самодеятельность здесь приводит к браку или поломке.
  • Номер позиции совпадает с картой наладки
  • Тип и диаметр инструмента правильные
  • Вылет инструмента — промерен и занесён в таблицу коррекций
  • Пластины/режущая часть — без сколов и явного износа
  • Инструмент нормально затянут в державке, биение проверено (если есть индикатор)
  • Коррекция радиуса введена если используется (G41/G42)

По регламенту: периодичность замены — в карте наладки (обычно по числу деталей или минутам резания).

Внеплановая проверка если:

  • Звук обработки изменился — стал более низким, появился скрип
  • Размер начал «плыть» в одну сторону (инструмент изнашивается — деталь растёт или уменьшается)
  • Шероховатость резко ухудшилась
  • Стружка стала другой — длиннее, другого цвета, налипает на инструмент
  • Нагрузка на шпиндель выросла на 20–30% от обычной

Допустимо: снизить подачу до 80–100% от программной при первой детали, если что-то настораживает. Поднять до 110–120% если всё стабильно, материал идёт хорошо и карта это допускает.

Нельзя: задирать обороты выше ограничения инструмента (написано на хвостовике или в каталоге), снижать обороты настолько что инструмент «тупит» — это хуже, чем быстро.

💡 Любые финальные режимы, которые прижились — записать в карту наладки. Иначе следующая смена начнёт заново.

Причины:

  • Новая пластина имеет другую геометрию посадки — нормально, нужно корректировать коррекцию
  • Пластина не до конца дожата в гнезде — вытащи, протри гнездо, установи снова
  • Новая фреза — другой реальный диаметр или вылет
  • Остатки стружки под пластиной (частая причина!)

Действие: промерь деталь, посчитай отклонение, введи корректирующий износный офсет. Не меняй геометрическую коррекцию — только износную (W или H offset зависимость от стойки).

  • Геометрическая коррекция (H, длина) — меняется только при замене инструмента или после промера на приборе. Не трогай «для подстройки».
  • Износная коррекция (W, D) — именно её используй для подстройки размера в серии
  • Считай правильно: деталь больше нужного → нужно убрать металл → коррекция в минус (для наружных поверхностей)

  • Записать фактические обороты и подачи если они отличаются от программных
  • Вложить в карту наладки операции
  • Передать сменщику устно + в журнале

📐 Контроль размеров и качества 8 вопросов

100% контроль (каждая деталь): размеры по карте контроля с пометкой «100%», критические посадочные поверхности, первая и последняя деталь партии.

Выборочно (по карте контроля): остальные размеры с указанной периодичностью — например каждые 5, 10 деталей.

Усилить контроль если: смена инструмента, смена заготовки (другая партия материала), нестабильная шероховатость, любые изменения в процессе.

Размер 25 +0.021/0: допустимо 25.000 до 25.021. Меньше 25.000 — брак. Больше 25.021 — брак.

Размер 25 ±0.1: допустимо 24.900 до 25.100.

Практическое правило: не доводи деталь до самой границы. Держи в середине поля допуска ± треть поля — это зона комфорта. Если размер ближе чем треть поля допуска к границе — пора корректировать коррекцию, не жди брака.

💡 Нет допуска на чертеже → смотри в общих технических требованиях чертежа, обычно IT12–IT14 по ГОСТ.
  • Первая деталь — 100% промер
  • Далее — по карте контроля (обычно каждые 5–20 деталей в зависимости от стабильности)
  • После каждой смены инструмента — промер сразу
  • Если размер начал «плыть» — участить до 100%
  • После перерыва в работе (обед, конец смены, остановка станка) — промер первой детали после перезапуска

Не жди пока выйдет за допуск. Действуй сразу:

  1. Определи тренд: размер уходит стабильно в одну сторону (износ инструмента) или скачет (нестабильный зажим, биение)
  2. Если монотонный уход — введи корректирующий износный офсет (маленький шаг, 0.02–0.05 мм)
  3. Промерь следующую деталь — убедись что размер вернулся к середине поля
  4. Если скачет — ищи нестабильность: зажим, биение, стружка под деталью

Нестабильность размера при нормальном инструменте — почти всегда причина в базировании или зажиме:

  • Стружка попадает под деталь между деталями — чисти каждую установку
  • Непостоянное усилие зажима (ручной зажим «на силу руки»)
  • Деталь «садится» по-разному на базовые поверхности — заусенцы на заготовке
  • Тепловой дрейф станка — первые детали после прогрева имеют другой размер
  • Разброс в партии заготовок — разный материал, разный припуск
  • Используй только износную (W/D) коррекцию, не геометрическую
  • Наружный диаметр/стенка больше → коррекция в «–» (убираем металл)
  • Отверстие меньше → коррекция в «+» (расширяем)
  • Вноси изменения пошагово: не более половины отклонения за один шаг
  • После каждого изменения — промер детали, потом следующий шаг
⚠️ Не кидайся сразу вносить всё отклонение — можно перескочить в другую сторону и получить брак.
  1. Отложить отдельно, пометить ярлыком «БРАК» — не смешивать с годными
  2. Записать: номер детали, операция, размер, фактическое значение, причина если ясна
  3. Уведомить мастера/технолога
  4. Не пытаться «исправить» брак самому без согласования
  5. Понять причину прежде чем продолжать серию — иначе следующие детали тоже в брак

🚨 Аварии и ошибки 7 вопросов
  • Нагрузка на шпиндель резко выросла (следи за монитором нагрузки)
  • Звук стал другим — глухой удар, скрежет, нерегулярные щелчки
  • Инструмент «жуёт» а не режет — размер уходит быстро
  • Координата одной из осей неожиданно изменилась (сбой нуля)
  • Заготовка сдвинулась в зажиме — слышно стук или видно визуально
  • Стружка длинная, не ломается, наматывается
⚠️ При любом из этих признаков — немедленно нажми «Подача Стоп», не дожидайся конца прохода.
  1. E-Stop или «Подача Стоп» — сразу
  2. Не трогай оси, не пытайся «вытащить» инструмент вручную двигая по G-коду
  3. Запиши номер аларма, название, время, на каком кадре произошло
  4. Сфотографируй экран стойки
  5. Вызови мастера/технолога — до его прихода ничего не меняй
  6. Осмотр: видимые повреждения инструмента, оснастки, шпинделя
  7. После осмотра — решение принимает мастер, не оператор
🚨 Никогда не пытайся самостоятельно отвести оси после столкновения — можно усугубить повреждения или потерять положение нуля.

💡 Хорошее описание аварии экономит часы диагностики. Плохое описание — «оно само» — ни о чём не говорит.

  • Запускать программу снова «проверить не повторится ли»
  • Скрывать аварию — это вопрос безопасности, не только экономики
  • Самостоятельно «подправить» погнутый инструмент молотком
  • Менять нули или коррекции до фиксации их текущих значений
  • Чек-лист перед каждым запуском — не по памяти, а по листу

  • Первая деталь — подача 10%, рука на «Стоп»

  • Передавай смену нормально: что было, что менял, что настораживало

Что проверить перед первым запуском

Тапни — отмечается. Все шаги должны быть зелёными перед CYCLE START. Пункты 💀 грабли — это реально случается и стоит дорого.

01 Станок и деталь до открытия программы
  • Референс (home) выполнен после включения — все оси в нуле станка
  • Деталь зажата надёжно, усилие не деформирует заготовку (тонкостенки — особо)
  • 💀 граблиЗона обработки чистая: убраны щупы, прихваты, инструменты измерения — причина №1 краша
  • Дверь станка закрыта, блокировки сработали (индикатор на пульте)
  • Давление воздуха / гидравлики зажима в норме (манометр)
02 Инструмент коррекции и вылет
  • Физически установленный инструмент совпадает с номером в программе (T1 = фреза Ø10, не сверло)
  • 💀 граблиДлина инструмента обмерена и внесена в таблицу: H-offset (Fanuc), длина в таблице инструментов (Siemens), TOOL.T (Heidenhain)
  • 💀 граблиFanuc: G43 H[номер] стоит до первого движения по Z — без него коррекция длины не активна
  • Вылет инструмента достаточен для глубины + зазора, хвостовик не упрётся в деталь
  • Инструмент надёжно зажат в патроне / оправке (момент затяжки)
  • Диаметр фрезы в таблице соответствует реальному (важно при коррекции радиуса G41/G42)
03 Нули и координаты привязка детали
  • 💀 граблиНулевая точка детали привязана верно и записана в нужную ячейку: G54 (Fanuc/Siemens), CYCL DEF 7.1 / preset (Heidenhain)
  • 💀 граблиZ-ноль: поверхность в программе совпадает с реальной точкой привязки — путаница «верх детали / верх тисков» бьёт в стол
  • Зазор R-плоскости (R / Q200) выше всех зажимных элементов и прихватов
  • Активная система координат в программе (G54–G59) совпадает с той, куда записан ноль
  • 💀 граблиОстаточное смещение предыдущей детали сброшено — G54 не содержит «чужой» ноль от прошлой программы
04 Программа синтаксис и логика
  • Программа прошла графическое моделирование (симуляция на стойке или в CAM)
  • 💀 граблиПервая строка сбрасывает всё: G40 G49 G80 (Fanuc), TRAFOOF CYCLE800() (Siemens 5ос.), CYCL DEF 0 (Heidenhain) — без этого «хвост» предыдущей программы активен
  • Единицы мм/дюймы: G21 или G20 (Fanuc), DIAMOF/DIAMON совпадает с заготовкой
  • 💀 граблиКоррекция радиуса (G41/G42 или RL/RR) выключается (G40/R0) до каждой смены инструмента и в конце программы
  • Все вызовы подпрограмм L / M98 / CALL имеют соответствующие файлы на стойке
  • Программа заканчивается M30 (Fanuc/Siemens) или END PGM (Heidenhain)
05 Первый прогон запуск и контроль
  • 💀 граблиOverride подачи на 5–10%, Override шпинделя снижен — первый проход всегда медленно
  • Первое движение по Z проверено в покадровом режиме (Single Block / Satzweise)
  • Рука у Feed Hold и аварийного стопа весь первый прогон
  • 💀 граблиСОЖ включена до входа инструмента в металл — сухой вход в жаропрочном / алюминии = наклёп или налипание
  • 💀 граблиПосле первой детали — замер. Коррекция до запуска серии, не после третьей
