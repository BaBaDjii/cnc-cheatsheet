Смещения и нули

Fanuc

Смещения нуля детали

6 стандартных смещений: G54–G59. Расширенные: G54.1 P1–P48 (или G54.1 P1–P300 в зависимости от опции).

Задаются через меню OFFSET / WORK или командой: G10 L2 P1 X... Y... Z...

Пример
G54          ; активировать смещение 1
G10 L2 P1 X-120.5 Y-85.0 Z0 ; записать G54 из программы
Siemens

Нулевые смещения (ZO)

G54–G57 + расширенные G505–G599 (FRAME-функции). Программируются через TRANS/ATRANS (трансляция) и ROT/AROT (поворот).

Возможна работа с составными FRAME: FRAME = CTRANS(X, v1, Y, v2)

Пример
G54              ; активировать нулевое смещение
TRANS X50 Y30    ; дополнительный сдвиг
ATRANS X10       ; накопительный сдвиг
Heidenhain

Нулевые точки (Preset / REF)

Используются таблицы нулевых точек (Preset-таблица) или цикл CYCL DEF 7. Активация через CYCL DEF 7.0 / 7.1 / 7.2 / 7.3.

Каждый параметр цикла задаёт смещение по одной оси.

Пример
CYCL DEF 7.0 НУЛЕВАЯ ТОЧКА
CYCL DEF 7.1 X+50
CYCL DEF 7.2 Y+30
CYCL DEF 7.3 Z+0
; или через таблицу:
CYCL DEF 7.0
CYCL DEF 7.1 #5  ; строка 5 таблицы

Коррекция инструмента

Fanuc

Таблица офсетов (H/D)

  • H-офсет — длина инструмента (G43 H...)
  • D-офсет — радиус инструмента (G41/G42 D...)
  • Активируется через G43: G43 H1
  • Отмена: G49
  • Износ хранится отдельно в колонке Wear
Пример вызова
T01 M06          ; сменить на инструмент 1
G43 H01 Z100.   ; активировать коррекцию длины
G41 D01 X...    ; коррекция радиуса влево
Siemens

Таблица инструментов TO

  • Коррекция автоматически при вызове T + M6
  • D1, D2… — режущие кромки одного инструмента
  • Длина по Z задаётся как Length1 в таблице
  • Радиус — Radius в таблице TO
  • Смена кромки: D2 (без смены T)
Пример вызова
T1 D1            ; инструмент 1, кромка 1
M6               ; смена инструмента
G41              ; коррекция радиуса влево (из таблицы)
Heidenhain

Таблица инструментов TOOL.T

  • Все данные — в файле TOOL.T (Length, Radius, DL/DR — износ)
  • Вызов: TOOL CALL n Z S5000
  • Коррекция радиуса: RL / RR / R0 в кадре движения
  • Отдельный файл TOOL_P.T для карманов
  • Ось инструмента задаётся при вызове: X / Y / Z
Пример вызова
TOOL CALL 5 Z S6000  ; инструмент 5, ось Z, 6000 об/мин
L X+50 Y+30 RL F300  ; движение с коррекцией радиуса влево

Опасные различия — читать внимательно!

⚠

Компенсация длины инструмента

На Fanuc коррекция длины не активна без явного G43 H.... На Siemens и Heidenhain она применяется автоматически при вызове инструмента. Если перенести программу Fanuc на Siemens без удаления G43 — программа может не запуститься или отработать с ошибкой.

⚠

Единицы: G20/G21 vs G70/G71

Fanuc: дюймы = G20, метрика = G21. Siemens: дюймы = G70, метрика = G71. Если перепутать — инструмент уйдёт в 25 раз дальше или ближе нужной позиции. Катастрофический риск столкновения.

⚠

Отмена циклов сверления

Fanuc: цикл продолжает действовать до явного G80. Siemens и Heidenhain: цикл однократный или требует MCALL. Без G80 на Fanuc любое движение XY может вызвать незапланированное сверление.

⚠

Вызов подпрограммы (M98/CALL)

Fanuc: M98 P1001 L3 (номер + повтор). Siemens: SUB1() или CALL "SUB1". Heidenhain: CALL PGM "путь". При переносе программы подпрограммы просто не вызовутся без ошибки — опасная ситуация.

⚠

Направление дуги у Heidenhain

Heidenhain использует DR+ (против часовой) и DR- (по часовой) — это обратная логика по сравнению с G02/G03. При конвертации постпроцессора легко перепутать направление контурной фрезеровки.

⚠

Смена инструмента: T и M06

Fanuc: T01 M06 — сначала подготовка (T), потом физическая смена (M06). Если написать только T01 без M06 — инструмент не сменится. На Siemens поведение зависит от конфигурации PLC — может сменяться при T или при M6.

⚠

Конец программы: M30 vs END PGM

Heidenhain требует строку END PGM имя MM в конце каждой программы. Без неё программа не верифицируется. Fanuc/Siemens: M30 — обязательно для сброса указателя. Без M30 на Fanuc программа может запуститься повторно с середины.

⚠

Инкрементальный режим Heidenhain

Heidenhain: для инкрементального перемещения используется префикс I перед адресом оси: L IX+10. Fanuc/Siemens: переключение режима через G91. Смешивание логик — частая ошибка при конвертации программ.

Что проверить перед первым запуском

Нажимайте на каждый пункт для отметки. Все галочки должны быть зелёными перед нажатием CYCLE START.

🔧 Инструмент

  • Номер инструмента в программе совпадает с физически установленным
  • Длина и радиус инструмента внесены в таблицу (H/D на Fanuc, TOOL.T на Heidenhain)
  • Fanuc: G43 H... присутствует перед первым движением по Z
  • Инструмент надёжно зажат, вылет соответствует глубине обработки
  • Коррекция радиуса (G41/G42 или RL/RR) выключается до конца программы

📍 Нули и координаты

  • Нулевая точка детали привязана верно (G54 или CYCL DEF 7)
  • Ось Z: ноль на поверхности детали или на торце — соответствует программе
  • Зазор R-плоскости (R5 или Q200) больше зажимных элементов
  • Система координат (G17/G18/G19) соответствует ориентации детали

⚙️ Программа

  • Первая строка содержит G40 G49 G80 (Fanuc) или сброс смещений
  • Единицы измерения (G20/G21 или G70/G71) соответствуют программе
  • Программа прошла графическое моделирование (sim / проверка траектории)
  • Программа заканчивается M30 (Fanuc/Siemens) или END PGM (Heidenhain)
  • Подпрограммы присутствуют и вызываются правильным синтаксисом

🏭 Станок и деталь

  • Деталь надёжно зажата, усилие зажима не деформирует заготовку
  • Зона обработки свободна — нет прихватов, болтов, измерительных щупов
  • Первый запуск — подача Override на 5–10%, скорость Override снижена
  • Рука у кнопки Feed Hold / аварийного стопа при первом прогоне
  • СОЖ включена (если требуется), давление в норме
